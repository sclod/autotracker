# QA Checklist (ручные проверки)

## A. Общие
- `npm i` → `npm run lint` → `npm run build` проходят без ошибок.
- В `.env` установлено `SEED_DEMO=true` (для демо‑номеров `123456/654321`).
- На мобиле нет горизонтального скролла.

## B. Публичная часть
- `/`: header виден и кликабелен поверх hero‑видео.
- Видео `main_video.mp4` грузится, на мобиле высота не ломается.
- Секция марок: переключение EU/CN/KR работает, карточки ведут на `/catalog/...` без 404.

## C. Каталог
- `/catalog` открывается.
- `/catalog/eu` → `/catalog/eu/bmw` → `/catalog/eu/bmw/any-id` открываются, breadcrumb корректный.

## D. Лиды
- Отправка LeadForm с главной/каталога создаёт запись.
- `/admin/leads` показывает лиды (новые сверху).

## E. Трекер
- `/track` форма работает, редиректит в `/track/{trackingNumber}`.
- `/track/123456` и `/track/654321` показывают данные.
- Таблица этапов корректно отображает статусы/дату/коммент.

## F. P1: Чат / Карта / Файлы
- Чат: клиент отправил → админ увидел → админ ответил → клиент увидел (polling).
- Карта: точки отображаются, current видно.
- Файлы: админ загрузил → файл появился в списке → клиент видит файл → скачивание/просмотр работает.

## G. Security (локально)
- `/admin` без логина редиректит на `/admin/login`.
- Неверный пароль не логинит.

# Smoke Tests

## Установка Playwright (один раз)
`npx playwright install`

## Запуск
`npm run test:smoke`

## Переменные окружения (опционально)
- `SMOKE_TRACKING_NUMBER` — номер демо‑заказа (по умолчанию `123456`)
- `SMOKE_ACCESS_CODE` — код доступа (по умолчанию `111111`)
- `ADMIN_PASSWORD` — пароль админа (по умолчанию `change-me`)
