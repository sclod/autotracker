# QA Checklist (ручные проверки)

## A. Общие
- `npm i` → `npm run lint` → `npm run build` проходят без ошибок.
- В `.env` установлено `SEED_DEMO=true` (если нужны демо-номера `123456/654321`).
- Нет горизонтального скролла на ключевых страницах.
- После изменения cookie path — перелогиниться в админке.

## B. Публичная часть
- `/`: header виден и кликабелен поверх hero-видео.
- `main_video.mp4` грузится, на мобиле высота не ломается.
- Секция марок: переключение USA/EU/China работает, карточки ведут на `/catalog/{region}/{make}` без 404.

## C. Каталог (Slim)
- `/catalog/usa`, `/catalog/eu`, `/catalog/china` открываются, по 10 карточек.
- Детальная: `/catalog/{region}/{make}/{id}` открывается, галерея/CTA на месте.
- CTA “Хочу такой автомобиль” префиллит LeadForm (message содержит модель + регион).
- Редиректы: `/catalog/cn` → `/catalog/china`, `/catalog/kr` и `/catalog/korea` → `/catalog/usa`.
- `/catalog` (хаб) отключён по требованию.

## D. Лиды
- Отправка LeadForm с главной/каталога создаёт запись.
- `/admin/leads` показывает лиды (новые сверху).
- Конвертация лида в заказ работает, после — кнопки “Скопировать ссылку/код”.

## E. Трекер
- `/track` форма работает, редиректит в `/track/{trackingNumber}`.
- `/track/123456` и `/track/654321` показывают данные (если SEED_DEMO=true).
- Этапы отображают статус/дату/комментарий корректно.

## F. P1: Чат / Карта / Файлы
- Чат: клиент отправил → админ увидел → админ ответил → клиент увидел (polling).
- Карта: точки отображаются, current видно.
- Файлы: админ загрузил → файл в списке → клиент видит → скачивание работает.

## G. Security (локально)
- `/admin` без логина редиректит на `/admin/login`.
- Неверный пароль не логинит.
- Rate-limit не ломает нормальный сценарий, но блокирует спам.

## H. Мобилка / брейкпоинты
- 375 / 768 / 1024 / 1440: нет перекрытий, нет прыжков контента.
- Hero: CTA виден, header не перекрывает кнопку.
